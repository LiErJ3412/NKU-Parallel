# PCFG Guess + MD5 SIMD 实验结果报告

## 1. 测试环境与数据

- 系统：macOS Darwin arm64
- 测试数据：`../../guess_data/rockyou.txt`
- 生成上限：超过 10,000,000 条 guesses 后停止
- 最终生成数量：10,106,852 条 guesses
- 实际参与哈希数量：9,528,822 条 passwords，分 9 个 batch 处理

## 2. 编译方式

SIMD 版本：

```bash
g++ -std=c++17 -O2 main.cpp train.cpp guessing.cpp md5.cpp -o main
```

串行哈希版本：

```bash
g++ -std=c++17 -O2 -DUSE_SERIAL_HASH main.cpp train.cpp guessing.cpp md5.cpp -o main_serial
```

其中 `main` 使用 `MD5HashSIMD` 批量哈希，`main_serial` 使用 `MD5Hash` 对每个口令逐条哈希。

## 3. 普通运行结果

| 版本 | Guess time | Hash time | Train time | real time |
|---|---:|---:|---:|---:|
| SIMD `main` | 0.207360 s | 0.972832 s | 7.64131 s | 10.06 s |
| 串行 `main_serial` | 0.198864 s | 1.31642 s | 7.39758 s | 10.08 s |

基于普通运行结果：

- 哈希阶段加速比：`1.31642 / 0.972832 = 1.35x`
- 哈希耗时下降：约 `26.1%`
- 总运行时间变化不明显，因为训练阶段约 7.4 到 7.6 秒，占总时间主要部分，且这部分没有被 SIMD 哈希优化加速。

## 4. 低开销 Profile 结果

| 阶段 | SIMD | 串行 |
|---|---:|---:|
| `model::train` | 5.98724 s | 6.08221 s |
| `model::order` | 1.71679 s | 1.76831 s |
| `PriorityQueue::init` | 0.614062 s | 0.737306 s |
| `PriorityQueue::PopNext` | 0.212006 s / 516 calls | 0.189324 s / 516 calls |
| Hash batches | 1.00891 s / 9 batches | 1.32423 s / 9 batches |
| Guess + Hash total | 1.23040 s | 1.52299 s |

基于低开销 profile：

- 哈希阶段加速比：`1.32423 / 1.00891 = 1.31x`
- Guess + Hash 阶段整体加速比：`1.52299 / 1.23040 = 1.24x`
- 哈希耗时下降：约 `23.8%`

## 5. 函数热点分析

macOS 上没有 Linux `perf`。尝试使用 `sample` 时被系统权限限制，`xctrace` 需要完整 Xcode，因此最终采用两类方式：

1. 低开销手动计时：统计训练、排序、队列初始化、`PopNext`、哈希 batch 等关键阶段。
2. 全函数插桩：使用 `-finstrument-functions` 做辅助热点观察。该方法开销很大，不用于计算加速比。

全函数插桩观察到的主要热点包括：

- `model::FindPT(PT)`：训练/排序相关热点，调用频繁。
- STL 容器和字符串操作：`vector::size`、`vector::operator[]`、`string::data` 等调用量很大。
- SIMD MD5 内部函数：`Add4`、`RotateLeft4`、`HH4`、`MD5HashSIMD` 等。

## 6. 结论

SIMD 优化主要作用在 MD5 哈希阶段。实验中哈希阶段加速约 `1.31x` 到 `1.35x`，对应哈希耗时下降约 `24%` 到 `26%`。

整体程序加速不明显，原因是训练、模型排序和 guess 生成占据了较大比例，而这些阶段没有被当前 SIMD 哈希优化覆盖。若继续优化整体效率，优先方向应包括：

- 优化 `model::FindPT` 等频繁查找函数。
- 减少 `vector` / `string` 高频小操作和拷贝。
- 并行化 `PriorityQueue::Generate` 中生成 guesses 的循环。
- 降低训练阶段输出和解析开销。
